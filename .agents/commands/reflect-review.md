# Reflect Review

上面的报告是审计者静态阅读代码、代码 Diff、搜索网络资料后给出的建议清单。

建议可能是真问题、也可能是幻觉。

而你，是一个在真实代码仓库里工作的 Agent。

你的任务是「逐条深度搜索与深度思考 + 给出你的判断 + 给出最小修复建议」。

## 输出要求

### 你判断的证据（Data）

### 你判断的担保（Warrant）

### 你判断的结论（Claim）

1. 真实
2. 可能存在问题，你的知识不足以判断，你对问题的上下文不足以判断
3. 幻觉
4. 风格风格问题 / 过度优化问题

### 你判断的影响等级

1. 会崩 / 严重漏数 / 污染数据
2. 高概率错误
3. 边角 / 低概率
4. 纯优化

### 你的建议动作

- 若为真实问题：给「最小风险修复方案」（prefer small patch over refactor），并指出可能副作用
- 若为错误/过度优化：说明为什么不改更好
- 若你无法判定：把「不确定点」列入文末 Questions，并给出你需要我提供什么信息才能定案

## 工具参考

- 运行代码协助你判断
- 运行 Linter 或 Tester 协助你判断
- 加载 Agent Skills 协助你判断
- 阅读源代码（比如在 .venv 的 Python 源代码）协助你判断
- 咨询神谕协助你判断
- 咨询图书管理员协助你判断
- 咨询子智能体协助你判断
